# VPN Shop — Telegram-бот продажи VLESS VPN + веб-страница подписки

Telegram-бот для продажи VPN-подписок на базе VLESS-сервера
([документация по установке сервера](https://vlesshelp.h1cloud.net/)) и
веб-страница подписки в стиле Remnawave: статус, ссылка подписки, QR-код и
инструкции по подключению для iOS / Android / Windows / macOS / TV.

Стек: Python 3.11+ (работает на 3.13), aiogram 3, SQLAlchemy 2.0 (async, SQLite),
FastAPI + uvicorn, aiohttp, Jinja2, segno.

## Возможности

**Бот:**

- Продажа тарифов (срок / лимит трафика / лимит устройств / цена), тарифы задаются в `config.yml`.
- Оплата: **Telegram Stars** (работает из коробки), **CryptoBot** (Crypto Pay API), **с внутреннего баланса**.
- Пополнение баланса: быстрые суммы из конфига + произвольная сумма.
- Пробный период (включается в конфиге: срок, трафик, устройства; один раз на пользователя).
- Реферальная программа: процент с каждого платежа приглашённого зачисляется на баланс пригласившего.
- Промокоды двух типов: пополнение баланса и дни подписки (ограничение по числу использований и сроку действия).
- Напоминания об окончании подписки за 3 дня и за 1 день + уведомление об истечении, с кнопкой продления.
- Продление: повторная покупка **продлевает** текущую подписку (дни прибавляются к остатку).
- Мультилокации: один мастер-сервер с федерацией или несколько независимых серверов с выбором локации при покупке.
- Админ-панель: статистика, рассылка, управление пользователями (дни, баланс, бан), промокоды, последние платежи, уведомления о покупках.

**Веб-страница подписки** (`https://ваш-домен/{uuid}`):

- Статус подписки, дней до окончания, дата окончания (МСК), трафик, лимит устройств.
- Ссылка подписки с копированием в один клик и QR-код.
- Инструкции по платформам с deep-link кнопками: Happ, Streisand, Shadowrocket, V2RayTun, Hiddify, v2rayN, FlClash, FoXray, NekoBox.
- Прокси эндпоинтов подписки через ваш домен: `/{uuid}/sub`, а также `/{uuid}/raw|json|clash|sing-box`.

## Архитектура

Проект состоит из трёх частей:

```
 Telegram ──► бот (python -m app.bot.main) ──┐
                                             ├──► SQLite (data/shop.db)
 Браузер/приложение ──► веб (uvicorn) ───────┘
                              │
                              ▼ (прокси /sub)
                    VLESS-сервер(ы) — API + sub-порт
                    (установка: https://vlesshelp.h1cloud.net/)
```

- **Бот** — приём платежей, выдача/продление клиентов на VLESS-сервере через его API
  (`/create`, `/edit`, ban/unban и т.д.), напоминания, админка.
- **Веб** — страница подписки и прокси подписочных ссылок. Приложениям выдаётся ссылка
  вида `{PUBLIC_BASE_URL}/{uuid}/sub` — она ведёт на ваш домен с нормальным TLS-сертификатом,
  а веб-сервис уже сам ходит на sub-порт VLESS-сервера (у которого сертификат самоподписанный).
- **VLESS-сервер** — ставится отдельно по [официальной документации](https://vlesshelp.h1cloud.net/).
  Боту нужны: адрес API-порта, API-токен и адрес sub-порта.

Бот и веб — два отдельных процесса, использующие общую базу SQLite и общий конфиг.

## Требования

1. Установленный и работающий **VLESS-сервер** по документации
   [vlesshelp.h1cloud.net](https://vlesshelp.h1cloud.net/). После установки у вас должны быть:
   - адрес API (`https://IP:API_PORT`) и **API-токен**;
   - адрес sub-сервера (`https://IP:SUB_PORT`) — порт подписок.
2. **Токен Telegram-бота** от [@BotFather](https://t.me/BotFather).
3. **Домен**, направленный (A-запись) на сервер, где будет крутиться веб-часть —
   для страницы подписки и подписочных ссылок.
4. Docker + Docker Compose (для установки через Docker) либо Python 3.11+ (для ручной установки).

## Установка через Docker Compose

```bash
git clone https://github.com/welwes/video-seller.git vpn-shop
cd vpn-shop

# 1. Секреты
cp .env.example .env
nano .env          # BOT_TOKEN, ADMIN_IDS, PUBLIC_BASE_URL, при необходимости CRYPTOBOT_TOKEN

# 2. Структура магазина: название, тарифы, серверы, триал
cp config.example.yml config.yml
nano config.yml    # servers: api_url, api_token, sub_url; plans: тарифы и цены

# 3. Запуск бота и веб-части
docker compose up -d --build

# Логи
docker compose logs -f bot
docker compose logs -f web
```

После запуска:

- бот отвечает на `/start` в Telegram;
- веб слушает `127.0.0.1:8080` на хосте (наружу его выводит Caddy — см. ниже — либо ваш собственный reverse proxy).

База данных лежит в `./data/shop.db` (том, переживает пересборку контейнеров).

Обновление:

```bash
git pull
docker compose up -d --build
```

## Домен и HTTPS (Caddy)

В комплекте есть сервис `caddy` (профиль `proxy`): он слушает 80/443, автоматически
получает и продлевает сертификат Let's Encrypt и проксирует домен на веб-сервис.

1. Направьте A-запись домена (например, `sub.example.com`) на IP сервера.
2. Добавьте домен в `.env` (docker compose прочитает переменную оттуда):

   ```dotenv
   DOMAIN=sub.example.com
   PUBLIC_BASE_URL=https://sub.example.com
   ```

   `PUBLIC_BASE_URL` и `DOMAIN` должны указывать на один и тот же домен:
   именно из `PUBLIC_BASE_URL` бот собирает ссылки на страницу подписки.

3. Запустите с профилем `proxy`:

   ```bash
   docker compose --profile proxy up -d
   ```

Если у вас уже есть свой nginx/traefik/Caddy на хосте — профиль `proxy` не включайте,
просто проксируйте домен на `127.0.0.1:8080`.

**Зачем вообще проксировать подписку через свой домен?** Sub-порт VLESS-сервера отдаёт
подписку по HTTPS с **самоподписанным сертификатом** — многие приложения (особенно на iOS)
отказываются добавлять такую ссылку. Веб-сервис отдаёт ту же подписку по адресу
`https://ваш-домен/{uuid}/sub` уже с нормальным сертификатом, а к VLESS-серверу ходит сам,
игнорируя самоподписанный сертификат (`verify_ssl: false` в конфиге).

## Установка без Docker (venv + systemd)

```bash
# Пользователь и код
sudo useradd -r -m -d /opt/vpn-shop -s /usr/sbin/nologin vpnshop
sudo -u vpnshop git clone https://github.com/welwes/video-seller.git /opt/vpn-shop/app
cd /opt/vpn-shop/app

# Окружение
sudo -u vpnshop python3 -m venv /opt/vpn-shop/venv
sudo -u vpnshop /opt/vpn-shop/venv/bin/pip install -r requirements.txt

# Конфиги
sudo -u vpnshop cp .env.example .env        # заполнить
sudo -u vpnshop cp config.example.yml config.yml   # заполнить
```

Два процесса — два systemd-юнита.

`/etc/systemd/system/vpn-shop-bot.service`:

```ini
[Unit]
Description=VPN Shop — Telegram bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=vpnshop
WorkingDirectory=/opt/vpn-shop/app
ExecStart=/opt/vpn-shop/venv/bin/python -m app.bot.main
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/vpn-shop-web.service`:

```ini
[Unit]
Description=VPN Shop — subscription web page
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=vpnshop
WorkingDirectory=/opt/vpn-shop/app
ExecStart=/opt/vpn-shop/venv/bin/uvicorn app.web.main:app --host 0.0.0.0 --port 8080
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Запуск:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now vpn-shop-bot vpn-shop-web
journalctl -u vpn-shop-bot -f
```

Домен в этом варианте проксируйте на `127.0.0.1:8080` любым удобным способом
(Caddy на хосте, nginx + certbot и т.п.).

## Настройка локаций

Локации задаются списком `servers:` в `config.yml`. Поддерживаются две схемы.

### Схема 1: один мастер-сервер с федерацией

Если ваши VLESS-серверы объединены в федерацию (см. документацию сервера), подписка
мастер-сервера уже содержит **все** локации. В конфиге достаточно одной записи — пользователь
ничего не выбирает, а локации переключает прямо в приложении:

```yaml
servers:
  - key: "main"
    name: "🌍 Все локации"
    api_url: "https://1.2.3.4:56789"   # API мастер-сервера
    api_token: "MASTER_TOKEN"
    sub_url: "https://1.2.3.4:56790"   # sub-порт мастер-сервера
    verify_ssl: false
```

### Схема 2: несколько независимых серверов

Каждый сервер — отдельная локация; пользователь выбирает её при покупке.
Подписки на разных локациях независимы (у одного пользователя может быть по подписке
на каждую локацию, продлеваются отдельно):

```yaml
servers:
  - key: "nl"
    name: "🇳🇱 Нидерланды"
    api_url: "https://10.0.0.1:56789"
    api_token: "NL_TOKEN"
    sub_url: "https://10.0.0.1:56790"
    verify_ssl: false
  - key: "de"
    name: "🇩🇪 Германия"
    api_url: "https://10.0.0.2:56789"
    api_token: "DE_TOKEN"
    sub_url: "https://10.0.0.2:56790"
    verify_ssl: false
```

Важно про `key`: это стабильный идентификатор (только `a-z`, `0-9`, `_`), он сохраняется
в базе вместе с подписками. Не меняйте `key` у сервера, на котором уже есть подписки, —
иначе бот потеряет связь между локальными подписками и этим сервером. Название `name`
менять можно свободно.

После изменения `config.yml` перезапустите оба сервиса:
`docker compose restart` (или `systemctl restart vpn-shop-bot vpn-shop-web`).

## Платежи

### Telegram Stars

Работают из коробки — никаких токенов и регистраций не нужно. Цена в звёздах берётся
из `price_stars` тарифа; если там `0`, считается автоматически:
`ceil(price_rub * stars_per_rub)` (курс `stars_per_rub` задаётся в `config.yml`).

### CryptoBot (Crypto Pay)

Оплата криптовалютой через [@CryptoBot](https://t.me/CryptoBot):

1. Откройте [@CryptoBot](https://t.me/CryptoBot) в Telegram.
2. Меню → **Crypto Pay** → **Create App** — создайте приложение.
3. Скопируйте выданный **API-токен** и вставьте его в `.env`:

   ```dotenv
   CRYPTOBOT_TOKEN=12345:AAxxxxxxxxxxxxxxxxxxxxxxxx
   ```

4. Перезапустите бота.

Счета выставляются в рублях (`currency_type=fiat`), пользователь платит любой доступной
криптовалютой. Бот проверяет оплату по кнопке «Проверить оплату» и дополнительно сам
опрашивает висящие счета раз в минуту. Если `CRYPTOBOT_TOKEN` пуст — способ оплаты
просто не показывается.

### Внутренний баланс

Пополняется через меню «Пополнить баланс» (Stars или CryptoBot), промокодами типа
«баланс» и реферальными начислениями. Если баланса хватает на тариф, при покупке
появляется способ «С баланса».

## Админ-панель

Администраторы — Telegram ID из `ADMIN_IDS` в `.env`. Команда: `/admin`.

Возможности:

- **Статистика** — пользователи (всего / за сегодня), активные подписки, выручка
  (сегодня / месяц / всего), сводка по каждому VLESS-серверу (`/status`: число клиентов, онлайн).
- **Рассылка** — отправьте боту любое сообщение (текст, фото и т.п.), оно будет скопировано
  всем пользователям; в конце — отчёт «доставлено / не доставлено».
- **Пользователи** — поиск по ID или @username, карточка (баланс, подписки, платежи),
  действия: ± дни подписки, ± баланс, бан/разбан (бан также блокирует клиента на VLESS-серверах).
- **Промокоды** — список, создание, удаление (формат — ниже).
- **Платежи** — последние 15 платежей.

Уведомления о каждой покупке приходят всем администраторам автоматически.

## Промокоды

Создаются в админке (`/admin` → «Промокоды»). Формат строки создания:

```
КОД тип значение [макс_использований] [дней_действия]
```

- `тип` — `баланс` (зачислить рубли на баланс) или `дни` (добавить дни подписки);
- `макс_использований` — необязательно, `0` или пропуск = без ограничения;
- `дней_действия` — необязательно, сколько дней промокод активен с момента создания.

Примеры:

```
WELCOME баланс 100            # +100 ₽ на баланс, без ограничений
NY2026 дни 7 100 14           # +7 дней подписки, максимум 100 активаций, действует 14 дней
```

Каждый пользователь может активировать конкретный промокод только один раз.
Активация — через меню бота «Промокод».

## FAQ

**Почему ссылку подписки нельзя давать напрямую с VLESS-сервера?**
Sub-порт сервера использует самоподписанный TLS-сертификат — часть приложений
не примет такую ссылку. Поэтому подписка проксируется через ваш домен
(`https://ваш-домен/{uuid}/sub`) с валидным сертификатом от Let's Encrypt. Заодно
по адресу `https://ваш-домен/{uuid}` доступна страница с QR-кодом и инструкциями.

**Как изменить тарифы или цены?**
Отредактируйте `plans:` в `config.yml` и перезапустите сервисы. Поле `id` тарифа —
стабильный идентификатор: удаление тарифа не влияет на уже выданные подписки
(срок и лимиты уже применены на VLESS-сервере), просто тариф исчезнет из продажи.

**Что произойдёт при повторной покупке на той же локации?**
Подписка продлится: новые дни прибавятся к оставшемуся сроку
(или к текущему моменту, если подписка уже истекла), лимиты обновятся по новому тарифу.

**Как отключить пробный период / поменять его условия?**
Секция `trial:` в `config.yml` (`enabled: false` — выключить). Пробный период выдаётся
один раз на пользователя и это запоминается в базе.

**Можно ли запустить только веб или только бота?**
Да, это независимые процессы: `docker compose up -d web` / `docker compose up -d bot`.
Но для продажи нужны оба: бот выдаёт подписки, веб отдаёт страницу и ссылку подписки.

**Где лежат данные и что бэкапить?**
`./data/shop.db` (SQLite) плюс ваши `config.yml` и `.env`. Клиенты VPN живут на самих
VLESS-серверах — база бота хранит связь «пользователь Telegram → клиент на сервере»,
платежи, балансы и промокоды.

**Бот пишет «Сервер временно недоступен».**
Бот не смог достучаться до API VLESS-сервера. Проверьте `api_url` / `api_token` в
`config.yml`, доступность API-порта с машины, где запущен бот
(`curl -k https://IP:API_PORT/api/health`), и логи: `docker compose logs -f bot`.
